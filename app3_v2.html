Lesson 1: Introduction to Abstraction

From Functions to Classes and Object Oriented Programming (OOP)

Classes and OOP are used by software engineers to write and share code from each other.

OOP is a programming paradigm based on the concept of "objects", which are data structures that contain data, in the form of fields, often known as attributes; and code, in the form of procedures, often known as methods. A distinguishing feature of objects is that an object's procedures can access and often modify the data fields of the object with which they are associated.

By using OOP concept, it is much easier for software engineers to modify and update their code. It also helps reducing duplicate code.

Define _init_

_init_ initializes or creates space in memory. It works as the constructor.

Instance Variables.

All the variables associated with a specific instance are called instance variables.

Instance Methods

All of the functions inside the class that are associated with an instance and have the first argument as self are called instance methods.

Class Variables

The variables associated with class are called class variables.

Information about pre-defined class variables can be found in Pre-defined variables.

Inheritance

To reuse code, class child can inheritance from class parent by using the arguments and methods in class parent.

Methods Overriding

The ability of a sub-class to override a method that it inherited from its parent class is called method overriding.

Abstract Thinking

In HTML we are able to give a lot of different code the same attributes by putting it into a div tag. Then we can make all things with that tag inherit the same attirubutes and CSS.

OOP can be used with the same basic principle. After we creat a class for a group of objects with same features, all of the objects in this class can share the attributes and methods defined in this class.
